1. OVERALL ASSESSMENT
DoubleTuple mirrors IntTuple (verified) and is consistent with FloatTuple.
Aggregate types correct: min/max/median -> double, sum -> double (IEEE; ->±Inf
on overflow), average -> double. In-scope implementations route uniformly
through N.min/N.max/N.median (aggregates) and N.equals (contains 1278 / equals
919), so there is no visible semantic split inside this module. No class-specific
actionable findings; family-wide items apply.

APPLICABLE FAMILY-WIDE FINDINGS (full detail in IntTuple.txt):
  - [F1] copyOf(double[]) generic self-type return -> CCE footgun. High.
  - [F2] no arity-agnostic get(int) accessor. Medium.
  - [F3] element-destructuring accept/map/filter stop at arity 3. High.
  - [F4] empty-tuple: sum->0.0, min/max/median/average throw. High. KEEP.
  - Null-callback IAE(forEach) vs NPE(accept/map/filter) split. Medium.
