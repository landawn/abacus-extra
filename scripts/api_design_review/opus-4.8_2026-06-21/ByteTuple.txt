API DESIGN REVIEW — ByteTuple
MODEL=opus-4.8  DATE=2026-06-21  EXECUTION=PARALLEL (orchestrator-verified)

1. OVERALL ASSESSMENT
ByteTuple is a faithful mirror of IntTuple (verified: same families, same param
order, same return-type pattern). Aggregate widening is correct: min/max/median
-> byte (an element), sum -> int, average -> double (so sum cannot overflow int
and needs no @throws — correct and intentionally different from IntTuple). No
class-specific actionable findings.

APPLICABLE FAMILY-WIDE FINDINGS (see IntTuple.txt for full detail; same shape in
ByteTuple at the mirrored locations):
  - [F1] copyOf(byte[]) generic self-type return -> CCE footgun. High.
  - [F2] no arity-agnostic get(int) accessor. Medium.
  - [F3] element-destructuring accept/map/filter stop at arity 3. High.
  - [F4] empty-tuple: sum->0, min/max/median/average throw. High. KEEP.
  - Null-callback IAE(forEach) vs NPE(accept/map/filter) split. Medium.

2.-6. No ByteTuple-specific naming / return / placement / missing / deprecation
findings beyond the family-wide items above. of(8)/of(9) are @Deprecated
consistently (healthy).
